Key Parameters

Common Parameters

Parameter Type Description
computeid string Unique identifier for a VM instance in GCP (e.g., "vm-customer-123")
projectId string Firestore document ID for a customer project/account
customerId string Customer account identifier (same as projectId in most contexts)
machineType string Friendly profile label (`fast`, `very fast`, `fastest`).

Data Structures

Machine / VM (collection: machines) 
{
  "id": "string",              // Firestore document ID
  "computeid": "string",       // GCP instance name
  "customerId": "string",      // Associated customer/project ID
  "zone": "string",            // GCP zone (e.g., "us-central1-a")

  "status": "string",          // "RUNNING", "TERMINATED", "STOPPING"
  "creationTimestamp": "timestamp",
  "lastStartTimestamp": "timestamp",
  "lastStopTimestamp": "timestamp",
  "diskSizeGb": number,        // Boot disk size
  "networkIP": "string",       // Internal IP address
  "externalIP": "string",      // External IP (if assigned)
  "cpuPlatform": "string",     // CPU architecture

    "cpu": number,
    "memory": number,

  }
}

API Endpoints

  • createCompute

    Creates a new Google Compute Engine virtual machine associated with a customer record and tracked in Firestore. Requires mesh/project data uploaded beforehand via `uploadFile`.

    POST /createCompute

  • getComputeState

    Returns the latest recorded state for a compute workload by looking up its Google Compute Engine ID in Firestore.

    GET /getComputeState

  • getCredits

    Returns the token allocation for a given customer, including total tokens, tokens consumed, and computed balance.

    GET /getCredits

  • uploadFile

    Uploads mesh or project artifacts to Cloud Storage and ties them to a customer / project so `createCompute` can reference the returned `gs://` URLs.

    POST /uploadFile

  • getSimulationResults

    Provides the Cloud Storage link for the simulation output once the kernel uploads the result archive.

    GET /getSimulationResults

  • getMachinesByRegion Private

    Returns every Google Compute Engine VM in the project grouped by region with zones nested underneath. Private endpoint for internal tooling only.

    Private
    GET /getMachinesByRegion

  • getMachineCatalog Private

    Fetches every Compute Engine machine type available to the current project and groups them by region and zone. Private endpoint for internal tooling only.

    Private
    GET /getMachineCatalog

Kernels & Utilities

Calculation Kernel

Standalone executable that performs compute-intensive simulations on provisioned VMs. Takes project file and mesh file as parameters and produces result artifacts.

calculationKernel.exe --projectFile --meshFile

All documentation examples use C++ with libcurl and nlohmann/json
